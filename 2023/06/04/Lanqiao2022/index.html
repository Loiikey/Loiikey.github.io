D、最大数字

题目

给一个正整数 N

你可以对 N 的任意一位数字执行任意次以下 2 种操 作：

  1. 将该位数字加 1。如果该位数字已经是 9，加 1 之后变成 0。
  2. 将该位数字减 1。如果该位数字已经是 0，减 1 之后变成 9。

你现在总共可以执行 1 号操作不超过 A 次，2 号操作不超过 B 次。 请问你最大可以将 N 变成多少

G、故障

题目

每行表示一种故障原因，每一列表示一种故障现象。该矩阵表示故障原因 A 可能产生故障现象 22、33、44，故障原因 B 可能产生故障现象 11、33。

在实际开发过程中，如果出现了故障原因，工程师就可以根据故障现象，去计算每种故障原因产生的概率，并按照概率大小对故障原因进行排查，以达到快速定位故障原因的目的。

现在，我们假设系统开发中同一时间只会出现一种故障原因, 并且故障原 因引起各故障现象是独立事件。举个例子来说:

假设系统现在发生了故障原因 A, 有 1331 的概率出现故障现象 22，有 1441 的概率出现故障现象 33，有 1221 的概率出现故障现象 44。由于 33 种现象是独立发生的，因此有 12×3×42×3×41 的概率同时出现故障 22、33、44。

约定若相关性矩阵中没有 x 记号, 则表示该故障原因一定不会产生某故障现象，比如故障原因 A，一定不会产生故障现象 11。根据历史经验数据，我们统计得到了每一种故障原因出现的概率以及每一种故障原因对应的故障现象产生概率。

现在已知系统出现的故障现象，求问各个故障原因发生的概率。

思路

贝叶斯公式f(A|B) = f(AB) / f(B) = f(B|A) * f(A) / f(B) 发生B事件时 A事件发生的概率

注意下 发生要算 不发生也要算 不发生的概率就是 1 - 发生概率

题目

他发现机房里一共有 n 台电脑，编号为 1 到 n，电脑和电脑之间有网线连 接，一共有 n − 1 根网线将 n 台电脑连接起来使得任意两台电脑都直接或者间 接地相连。 小明发现每台电脑转发、发送或者接受信息需要的时间取决于这台电脑和 多少台电脑直接相连, 而信息在网线中的传播时间可以忽略。比如如果某台电脑 用网线直接连接了另外 d 台电脑，那么任何经过这台电脑的信息都会延迟 d 单 位时间 (发送方和接收方也会产生这样的延迟，当然如果发送方和接收方都是 同一台电脑就只会产生一次延迟)。 小明一共产生了 m 个疑问：如果电脑 ui 向电脑 vi 发送信息，那么信息从 ui 传到 vi 的最短时间是多少

I、齿轮

题目

他一共有 n 个齿轮，第 i 个齿轮的半径为 ri，他需要把这 n 个齿轮按一定 顺序从左到右组装起来，这样最左边的齿轮转起来之后，可以传递到最右边的 齿轮，并且这些齿轮能够起到提升或者降低转速 (角速度) 的作用。 小明看着这些齿轮，突然有 Q 个疑问：能否按一定顺序组装这些齿轮使得 最右边的齿轮的转速是最左边的齿轮的 qi 倍

思路

J、搬砖

题目

他一共有 n 块砖，他发现第 i 砖的重量为 wi，价值为 vi。他突然想从这些 砖中选一些出来从下到上堆成一座塔，并且对于塔中的每一块砖来说，它上面 所有砖的重量和不能超过它自身的价值。 他想知道这样堆成的塔的总价值（即塔中所有砖块的价值和）最大是多少。

思路

首先可以发现砖的摆放有一定顺序。

例如第 i 块砖放在第 j 块砖前面比第 j 块砖放在第 i 块砖前面好(我们将这里的好定义为重量 - 价值的最大值尽可能少)

max( - vi , wi - vj ) < max( - vj , wj - vi) wi - vj > - vj , wj - vi > - vi 所以的是 wi - vj < wj - vi 即 wi + vi < wj + vj 按照这个排序

具体可以看洛谷的国王游戏题目 和 acwing上的刷杂技的牛 这俩道题

排序之后按照01背包求解即可
